When it receives a GMP call (via axelarnet-gateway::Execute), it needs to ensure it is actually an ITS call, i.e it came from the trusted ITS edge contract on the source chain. This requires whitelisting the ITS addresses from each chain, similar to how each ITS edge contract is whitelisting trusted addresses. The axelarnet-gateway allows routing message to/from router as the other wasm gateway contract. But it also allows initiating messages to another chain, and executing messages that were received with for apps on Axelar. (edited)

The token deployer doesn’t need to be trusted if it’s deployed as a canonical ITS token. The minter role needs to be trusted if one is set. ITS factory allows the token deployer to set no minter. Hence, tokens deployed via the ITS factory can be trusted more than the flexible deployments from ITS contract. Please provide more context if your question is still unresolved.
